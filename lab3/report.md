# Лабораторная работа №3. Построение систем синтеза речи

## Цель работы

Получить практические навыки работы с системами синтеза речи на основе нейросетевых text-to-speech моделей с открытым исходным кодом.

## Выбранные модели

В работе сравниваются две open source модели из стека Coqui TTS:

1. **VITS** - современная end-to-end TTS-модель, объединяющая текстовый энкодер, механизм выравнивания, вариационную часть, normalizing flows и нейросетевой vocoder. Модель обучается непосредственно генерировать речь и не требует отдельной модели вокодера для инференса.
2. **Tacotron2** - классическая encoder-decoder модель с attention, которая преобразует текст в mel-спектрограмму. Для получения аудио из mel-спектрограммы используется open source neural vocoder, например HiFi-GAN.

Такое сравнение покрывает требование лабораторной: одна модель является современной, вторая относится к моделям из презентации.

## Датасет

Для обучения на русском языке нужен набор пар `текст-аудио`. Контрольные предложения из задания подходят для синтеза и оценки, но не подходят как обучающий датасет, потому что к ним нет записей голоса.

Основной вариант датасета:

- Mozilla Data Collective `Denis 1.0` или `Dmitri 1.0`;
- русский single-speaker TTS;
- около 2 часов речи;
- лицензия CC0;
- удобный размер для лабораторной работы.

Резервные варианты: `M-AILABS Russian` или `RUSLAN`, но их лучше ограничивать single-speaker subset, чтобы обучение было сопоставимым.

## Подготовка данных

Скрипт `src/prepare_dataset.py` приводит датасет к формату LJSpeech/Coqui:

- читает `metadata.csv`, `metadata.jsonl` или пары `sample.wav` + `sample.txt`;
- нормализует русский текст;
- приводит аудио к mono WAV и sample rate 22050 Гц;
- фильтрует слишком короткие и слишком длинные записи;
- формирует `metadata.csv`, `metadata_train.csv`, `metadata_val.csv`;
- сохраняет `dataset_summary.json`.

Команда:

```bash
python src/prepare_dataset.py \
  --raw-dir data/raw/denis \
  --output-dir data/processed/denis_ljspeech \
  --sample-rate 22050 \
  --val-ratio 0.05
```

## Эксперименты

Подготовлены три конфигурации:

| Эксперимент | Модель | Изменение |
| --- | --- | --- |
| `vits_baseline` | VITS | базовые параметры |
| `vits_lr_changed` | VITS | learning rate уменьшен с `0.0002` до `0.0001` |
| `tacotron2_baseline` | Tacotron2 | базовые параметры для сравнения с VITS |

Проверка конфигурации без запуска обучения:

```bash
python src/train.py --config configs/vits_baseline.json --dry-run
```

Запуск обучения после подготовки датасета:

```bash
python src/train.py --config configs/vits_baseline.json
python src/train.py --config configs/vits_lr_changed.json
python src/train.py --config configs/tacotron2_baseline.json
```

В этой версии лабораторной обучение не запускалось: скрипты и конфиги подготовлены для самостоятельного запуска.

## Логирование

Для анализа используются TensorBoard-логи из `runs/`:

- train/eval loss;
- learning rate;
- mel-спектрограммы;
- alignment plots для Tacotron2;
- компоненты loss для VITS: generator/discriminator, mel, KL, duration/feature loss, если они присутствуют в логах Coqui.

Запуск TensorBoard:

```bash

```

## Контрольный текст

Контрольные тексты сохранены в `texts/control_texts.json`. Они взяты из приложения к заданию и проверяют:

- фонетически представительный текст;
- вопросительную и восклицательную интонацию;
- двоеточие и тире;
- сокращения и аббревиатуры;
- числа и даты.

Пример синтеза после обучения:

```bash
python src/synthesize.py \
  --checkpoint checkpoints/vits_baseline/best_model.pth \
  --config checkpoints/vits_baseline/config.json \
  --texts texts/control_texts.json \
  --output-dir outputs/audio/vits_baseline
```

## Оценка качества

Скрипт `src/evaluate.py` считает:

- **MCD** - mel-cepstral distortion между референсом и синтезом;
- **WER/CER** - ошибки распознавания через внешнюю open source ASR-команду;
- длительность аудио и real-time factor, если синтез был выполнен через `src/synthesize.py`.

Пример:

```bash
python src/evaluate.py \
  --metadata data/processed/denis_ljspeech/metadata_val.csv \
  --synth-dir outputs/audio/vits_baseline \
  --output-dir outputs/metrics/vits_baseline
```

Для субъективной оценки используется mini-MOS по шкале 1-5:

- 1 - речь почти неразборчива;
- 2 - много артефактов, смысл часто теряется;
- 3 - речь разборчива, но заметно искусственная;
- 4 - хорошее качество с редкими дефектами;
- 5 - естественная речь без заметных дефектов.

## Ожидаемый анализ результатов

После обучения нужно сравнить:

- скорость сходимости по loss;
- устойчивость attention/alignment у Tacotron2;
- естественность и разборчивость синтеза;
- ошибки в чтении чисел, сокращений и аббревиатур;
- влияние уменьшения learning rate у VITS;
- скорость синтеза и real-time factor.

Ожидаемо VITS должен давать более стабильный end-to-end синтез и быстрее работать на инференсе. Tacotron2 может быть чувствительнее к качеству выравнивания и объёму данных, но полезен как классическая baseline-модель.

## Ответы на вопросы самопроверки

**1. Какие дополнительные характеристики можно подать на вход модели для улучшения точности синтеза речи?**

Можно подавать фонемное представление текста, ударения, speaker ID или speaker embedding, language ID, pitch/F0, energy, длительности фонем, темп речи, emotion/style/prosody embeddings, контекстные признаки и разметку пауз. Для русского языка особенно полезны фонемы, ударение и нормализация чисел/сокращений, потому что графемная запись часто неоднозначна.

**2. Что изменится в случае дообучения модели вместо обучения?**

При дообучении модель стартует с уже выученных представлений, поэтому обычно требуется меньше данных и времени, а качество на малом датасете выше. Но модель наследует ограничения исходного датасета: акцент, тембр, ошибки нормализации, bias и особенности фонемизатора. При слишком большом learning rate или малом датасете возможны переобучение и деградация ранее выученной речи.
