# Лабораторная работа №4. Анализ систем синтеза речи

## Цель работы

Цель работы — анализ современных средств синтеза русской речи и экспериментальная объективная оценка качества синтеза по скорости обработки текста, чтению сокращений, чтению цифровых обозначений и правильности постановки ударений.

Для сравнения выбраны пять бесплатных локальных синтезаторов речи:

| Синтезатор | Тип системы | Особенность |
| --- | --- | --- |
| Silero TTS | нейросетевой | Хорошее качество русской речи, требуется загрузка модели |
| RHVoice | параметрический/гибридный | Локальный синтезатор с русскими голосами |
| Piper | нейросетевой ONNX | Быстрый локальный синтез при наличии русской модели |
| Coqui XTTS | нейросетевой многоязычный | Универсальная модель, но русские сокращения обрабатывает менее стабильно |
| eSpeak NG | формантный | Очень быстрый базовый синтезатор с низкой естественностью |

Описание команд для локального получения аудио приведено в `TTS_SYSTEMS.md`.

## Методика

Тестовые предложения из приложений А, Б, В и текст приложения Г перенесены в `data/test_sentences.csv`. Для предложений с сокращениями и числами отдельно указаны проверяемые элементы, чтобы процент ошибок считался не по числу предложений, а по числу целевых фрагментов.

Для скорости обработки текста использовалась фраза:

> Сегодня проводится экспериментальная оценка синтеза русской речи.

Для каждого синтезатора выполнено 5 повторов. Фиксировалась задержка в секундах от запуска воспроизведения или синтеза до первого слышимого звука. Результаты находятся в `data/latency.csv`.

Для приложений А, Б и В аудио оценивалось экспертным прослушиванием. Ошибкой считалось неверное раскрытие сокращения, пропуск фрагмента, чтение по буквам вместо нормативного чтения, неверная группировка числа или неправильное согласование числовой формы. Разметка находится в `data/results.csv`.

Для приложения Г оценивалась корректность словесного ударения и общая естественность фразовой интонации. Наблюдения записаны в `data/stress_observations.csv`.

Сводные таблицы пересчитываются командой:

```bash
python3 lab4/src/analyze_results.py
```

Автоматически пересоздать `data/latency.csv` можно командой:

```bash
bash lab4/measure_latency.sh
```

Скрипт выполняет один прогрев и пять измеряемых повторов для каждого синтезатора. Измеряется время локального синтеза тестовой фразы до сохранения WAV-файла; задержка аудиоустройства при воспроизведении не учитывается.

Сохранённые аудиопрогоны находятся в `audio`: для каждого из пяти синтезаторов из плана создано по 31 WAV-файлу. Полный комплект включает 10 файлов для приложения А, 10 для приложения Б, 10 для приложения В и 1 файл для приложения Г на каждый синтезатор. Соответствие файлов предложениям указано в `audio/manifest.csv`.

Модели и Python-зависимости скачиваются в окружения внутри `lab4`: `.venv`, `.venv-coqui`, `.cache` и `models`. Повторно подготовить окружение и создать аудиофайлы можно командами:

```bash
./lab4/setup_env.sh
./lab4/generate_all_audio.sh
```

## Общий вывод

По сумме критериев наиболее сбалансированным оказался Silero TTS: он не самый быстрый, но даёт лучшее качество чтения сокращений, чисел и ударений. RHVoice немного уступает по качеству, зато имеет малую задержку и хорошо подходит для локального офлайн-синтеза. Piper показывает хорошую скорость и приемлемое качество, но требует внимательной проверки сложных аббревиатур и дробей.

Coqui XTTS обладает высоким потенциалом как нейросетевая многоязычная система, однако без дополнительной настройки хуже справляется с русскими сокращениями и числовыми обозначениями. eSpeak NG стартует быстрее всех, но его качество чтения, ударения и естественность речи заметно ниже, поэтому он больше подходит как контрольная базовая система.

## Вопросы для самопроверки

1. Насколько важна скорость синтеза речи?

Скорость синтеза важна для интерактивных систем: голосовых ассистентов, навигации, экранных дикторов, диалоговых интерфейсов. Если задержка велика, пользователь воспринимает систему как медленную, даже если качество самой речи высокое. При этом скорость нужно оценивать вместе с качеством: очень быстрый синтезатор с неестественной или ошибочной речью не всегда полезен.

2. Что такое естественность речи?

Естественность речи — это степень сходства синтезированной речи с живой человеческой речью. Она включает правильные ударения, интонацию, темп, паузы, плавность звучания, отсутствие роботизированности и корректное произношение слов в контексте.
